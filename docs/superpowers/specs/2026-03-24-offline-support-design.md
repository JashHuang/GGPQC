# V6 離線支援設計規格

**最後更新日期：** 2026-03-24

## 1. 設計目標

為「早安圖幫你做好 Web App」V6 版本新增離線支援能力，確保在網路不穩定或離線環境下仍能正常生成早安圖。

### 成功標準

- 使用者首次成功生成圖片後，背景圖會被快取
- 離線時仍可生成早安圖（使用快取的背景）
- 不影響原本的網路优先策略
- 不增加明顯的效能開銷

---

## 2. 技術方案

### 2.1 Service Worker 快取策略

**策略類型：** Stale-While-Revalidate（混合策略）

```
┌─────────────┐     ┌─────────────────┐     ┌─────────────┐
│   請求      │────▶│ Service Worker  │────▶│   網路      │
└─────────────┘     └─────────────────┘     └─────────────┘
                           │  │
                           │  │ 網路失敗
                           ▼  ▼
                    ┌─────────────┐
                    │   快取      │
                    └─────────────┘
```

**運作流程：**
1. 圖片請求發出 → Service Worker 攔截
2. 同時發起網路請求 + 讀取快取
3. 網路成功 → 回傳網路結果，同時更新快取
4. 網路失敗 → 回傳快取結果

### 2.2 快取容量管理

**限制條件：**
- 最大快取數量：20 張圖片
- 預估大小：10-20MB（每張圖 500KB-1MB @ 1080x1080）
- 淘汰策略：LRU（Least Recently Used）

**清理時機：**
- 新圖片入庫時，若已達上限，移除最舊的快取
- 每次 App 啟動檢查快取有效性

### 2.3 快取 Key 設計

```javascript
// 快取 Key 格式
const CACHE_NAME = 'gm-v6-bg-cache-v1';
const cacheKey = (url) => `bg-${hashUrl(url)}`;
```

## 3. 使用者體驗

### 3.1 首次使用提示

當 Service Worker 安裝成功且首次成功快取圖片後，顯示輕量提示：

> 「已為離線做好準備 ✓」

提示在 3 秒後自動消失，不打斷操作流程。

### 3.2 離線狀態指示

在完成頁加入離線狀態標示：

```
┌─────────────────────────┐
│ 🖼️ 使用快取的背景        │
│ 圖片可能與今天不同       │
└─────────────────────────┘
```

### 3.3 網路恢復

當網路恢復時，自動更新快取（背景圖）。

## 6. 測試情境

| 情境 | 預期行為 |
|------|---------|
| 離線 + 有快取 | 使用快取的背景圖生成 |
| 離線 + 無快取 | 顯示「需要網路」提示 |
| 網路恢復 | 正常生成並更新快取 |
| 快取已滿 | LRU 淘汰最舊圖片 |

---

## 7. 非功能性需求

- **效能影響：** 快取讀取 < 50ms，不影響生成速度
- **隱私：** 不上傳任何圖片或個人資料
- **向後相容：** 無 Service Worker 支援時正常降級
